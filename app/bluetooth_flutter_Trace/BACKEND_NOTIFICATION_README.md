# 后台通知功能说明

## 功能概述

为功率计报警功能添加了后台弹窗通知功能。现在当应用运行在后台时，如果设备触发报警，仍会通过系统通知显示弹窗，而不仅仅是震动。

## 实现内容

### 1. 添加了 flutter_local_notifications 依赖

- 在 `pubspec.yaml` 中添加了 `flutter_local_notifications: ^17.2.2` 依赖
- 在 `android/app/build.gradle.kts` 中启用了核心库去糖化支持

### 2. 创建了通知服务 (`lib/services/notification_service.dart`)

- 封装了 Flutter Local Notifications 插件
- 支持前台和后台通知显示
- 包含震动、声音等通知特性

### 3. 修改了报警服务 (`lib/services/alert_service.dart`)

- 集成了通知服务
- 前台时显示应用内通知和对话框
- 后台时始终显示系统通知弹窗

### 4. 添加了后台任务服务 (`lib/services/background_task_service.dart`)

- 监听应用生命周期变化
- 在后台模式下确保蓝牙扫描持续进行
- 管理后台通知权限

### 5. 更新了主应用配置 (`lib/main.dart`)

- 初始化通知服务和后台任务服务
- 请求必要的通知权限

## 测试方法

### 方法一：使用测试页面

1. 在应用中导航到 `/test-notification` 路由
2. 点击"测试前台通知"按钮 - 应显示应用内通知和对话框
3. 点击"测试后台通知"按钮 - 应显示系统通知弹窗
4. 将应用切换到后台，再次点击按钮测试后台通知

### 方法二：实际设备测试

1. 连接蓝牙设备并设置报警阈值
2. 将应用切换到后台运行
3. 等待设备触发报警（或手动触发）
4. 观察是否显示系统通知弹窗

## 注意事项

1. **权限要求**：

   - Android 13+ 需要通知权限
   - 需要振动权限用于震动提醒
   - 需要唤醒锁权限确保后台运行

2. **兼容性**：

   - Android 8.0+ 支持通知渠道
   - iOS 支持原生通知
   - Windows/Linux 暂不支持系统通知

3. **后台运行**：
   - Android 需要电池优化豁免
   - 应用需要在最近任务列表中保持活跃

## 故障排除

如果后台通知不工作：

1. 检查通知权限是否已授予
2. 确认应用未被电池优化限制
3. 检查设备是否支持通知功能
4. 查看日志输出中的调试信息

## 技术细节

- 使用 Flutter Local Notifications 插件实现跨平台通知
- 通过应用生命周期监听器检测前台/后台状态
- 集成原有的震动和声音报警机制
- 保持与现有报警逻辑的兼容性
