MVP 架构总览

项目的每个 UI 页面都由三个独立的类构成，严格遵循 MVP 模式，各层职责边界清晰、互不侵入。

Presenter（控制器）
继承 PageBase，实现全部生命周期回调；持有 View 和 Model 实例；处理用户输入事件；通过 _Manager->Push/Pop/Replace 驱动页面路由。
View（视图层）
仅负责 LVGL 控件的创建、布局、样式和动画时间线；通过公开的 ui.* 成员让 Presenter 直接操作控件；不持有任何业务逻辑。
Model（数据层）
创建 DataCenter Account，Subscribe 所需的数据发布者；通过 Pull 主动拉取数据，或通过 SetEventCallback 被动接收推送；通过 Notify 向发布者发送命令。

PageManager 路由机制

PageManager 维护一个 页面池（已注册的所有页面）和一个 页面栈（当前活跃的页面历史），所有路由操作最终都调用内部的 SwitchTo()。

四种路由操作

方法行为栈变化实际使用场景
Push(name) 压入新页面，旧页面保留在栈内（可缓存） 栈 +1 Dialplate → LiveMap / SystemInfos
Pop() 弹出当前页面，恢复上一页面 栈 -1 LiveMap 按下 slider 或 root 触发 LEAVE 事件
Replace(name) 替换栈顶页面，旧页面被强制卸载 栈不变 StartUp 2 秒计时器后替换为 Dialplate
BackHome() 强制清空页面栈，回到最底层页面 栈清空 从深层页面一键返回首页

Push 核心实现

bool PageManager::Push(const char* name, const PageBase::Stash_t* stash)
{
#ifdef MSC_USE_SD_CARD
    // 防止 USB 连接时进入 LiveMap（避免 SDIO 死锁）
    if (strcmp(name, "Pages/LiveMap") == 0 && HAL::USB_IsPlugged()) {
        PM_LOG_WARN("LiveMap blocked: USB connected");
        return false;
    }
#endif
    if (!SwitchAnimStateCheck()) return false;   // 动画进行中则拒绝
    if (FindPageInStack(name)) return false;     // 防止重复入栈
    PageBase* base = FindPageInPool(name);
    if (!base) return false;

    base->priv.IsDisableAutoCache = base->priv.ReqDisableAutoCache;
    _PageStack.push(base);
    return SwitchTo(base, true /*isEnterAct*/, stash);
}

Pop 核心实现

bool PageManager::Pop()
{
    if (!SwitchAnimStateCheck()) return false;
    PageBase* top = GetStackTop();
    if (!top) return false;

    if (!top->priv.IsDisableAutoCache) {
        top->priv.IsCached = false;  // 关闭自动缓存，页面将被卸载
    }
    _PageStack.pop();
    top = GetStackTop();             // 取出下一页面
    return SwitchTo(top, false /*isEnterAct*/, nullptr);
}

页面生命周期

每个页面由状态机驱动，共 8 个状态，对应 9 个可覆写的虚函数回调：

0
PAGE_STATE_IDLE — 页面已注册但未加载，对象存在于页面池中
1
PAGE_STATE_LOAD → 创建 LVGL 根对象 → 调用 onViewLoad()（创建 Model、View、注册事件）→ onViewDidLoad()
2
PAGE_STATE_WILL_APPEAR → 调用 onViewWillAppear()（设置焦点、弹出 Stash 参数、启动进入动画）→ 启动页面切换动画
3
PAGE_STATE_DID_APPEAR → 调用 onViewDidAppear()（创建周期定时器、注册 SD 卡事件等）
4
PAGE_STATE_ACTIVITY — 页面交互活跃期，事件回调持续触发
5
PAGE_STATE_WILL_DISAPPEAR → 调用 onViewWillDisappear()（清理焦点组、删除定时器、触发退出动画）
6
PAGE_STATE_DID_DISAPPEAR → 调用 onViewDidDisappear()（注销回调、释放 Model）→ 若已缓存则跳回 WILL_APPEAR，否则进入 UNLOAD
7
PAGE_STATE_UNLOAD → 调用 onViewUnload()（删除 View LVGL 对象、释放 Stash）→ onViewDidUnload() → 异步删除根对象 → 回到 IDLE

缓存机制

设置行为用途
默认（自动缓存） 第一次加载后保留 LVGL 对象，Pop 后直接从 WILL_APPEAR 恢复 大多数频繁访问的页面
SetCustomCacheEnable(false) 每次 Pop 后都完整执行 UNLOAD，下次重新加载 Startup（一次性）、LiveMap（实时刷新）
SetCustomAutoCacheEnable(false) 由开发者手动控制缓存使能 需精确控制内存的场景

DataCenter 数据总线机制

DataCenter 是全局单例数据总线，所有数据生产者和消费者以 Account（账户） 形式接入，通过发布-订阅模式实现完全解耦的数据流通。

Account 三种数据交互模式

方法方向描述典型用途
account->Pull("GPS", &gps, sizeof(gps)) 订阅者 → 发布者 主动拉取：订阅者按需读取发布者最新数据 SystemInfosModel 读取 GPS/MAG/IMU 一次性展示
SetEventCallback(onEvent) 发布者 → 订阅者（推送） 被动接收：发布者发布时回调订阅者 onEvent DialplateModel 被动接收 SportStatus 实时更新
account->Notify("Recorder", &cmd, sizeof(cmd)) 订阅者 → 发布者（命令） 通知：订阅者向发布者发送控制命令 DialplateModel 控制 Recorder 开始/暂停/停止

Account 工作原理（以 DialplateModel 为例）

// Model 初始化：创建账户并建立订阅关系
void DialplateModel::Init()
{
    // 账户名 "DialplateModel"，缓冲区大小 0（不发布数据，只订阅）
    account = new Account("DialplateModel", DataProc::Center(), 0, this);

    account->Subscribe("SportStatus");  // 接收运动数据（被动推送）
    account->Subscribe("Recorder");     // 控制录轨（Notify 命令）
    account->Subscribe("StatusBar");    // 控制状态栏（Notify 命令）
    account->Subscribe("GPS");          // 拉取 GPS 状态（Pull）
    account->Subscribe("MusicPlayer");  // 触发音效（Notify 命令）

    account->SetEventCallback(onEvent); // 注册推送接收回调
}

// 被动推送回调：SportStatus 发布者每秒发布一次时自动触发
int DialplateModel::onEvent(Account* account, Account::EventParam_t* param)
{
    if (param->event != Account::EVENT_PUB_PUBLISH) {
        return Account::RES_UNSUPPORTED_REQUEST;
    }
    if (strcmp(param->tran->ID, "SportStatus") != 0) {
        return Account::RES_PARAM_ERROR;
    }
    // 将发布者推送的数据复制到本地缓存
    DialplateModel* self = (DialplateModel*)account->UserData;
    memcpy(&self->sportStatusInfo, param->data_p, param->size);
    return Account::RES_OK;
}

// 主动拉取：检查 GPS 卫星数
bool DialplateModel::GetGPSReady()
{
    HAL::GPS_Info_t gps;
    if (account->Pull("GPS", &gps, sizeof(gps)) != Account::RES_OK)
        return false;
    return (gps.satellites > 0);
}

// Notify 命令：控制录轨模块
void DialplateModel::RecorderCommand(RecCmd_t cmd)
{
    DataProc::Recorder_Info_t recInfo;
    DATA_PROC_INIT_STRUCT(recInfo);
    recInfo.cmd = (DataProc::Recorder_Cmd_t)cmd;
    recInfo.time = 1000;
    account->Notify("Recorder", &recInfo, sizeof(recInfo));

    // 同时更新状态栏显示
    DataProc::StatusBar_Info_t statInfo;
    DATA_PROC_INIT_STRUCT(statInfo);
    statInfo.cmd = DataProc::STATUS_BAR_CMD_SET_LABEL_REC;
    switch (cmd) {
    case REC_START:
    case REC_CONTINUE: statInfo.param.labelRec.show = true;  statInfo.param.labelRec.str = "REC";   break;
    case REC_PAUSE:    statInfo.param.labelRec.show = true;  statInfo.param.labelRec.str = "PAUSE"; break;
    case REC_STOP:     statInfo.param.labelRec.show = false; break;
    }
    account->Notify("StatusBar", &statInfo, sizeof(statInfo));
}

// ── 录轨状态机 ──
void Dialplate::onRecord(bool longPress)
{
    switch (recState) {
    case RECORD_STATE_READY:
        if (longPress) {
            if (!Model.GetGPSReady()) { Model.PlayMusic("Error"); return; }
            Model.PlayMusic("Connect");
            Model.RecorderCommand(Model.REC_START);
            SetBtnRecImgSrc("pause");
            recState = RECORD_STATE_RUN;
        }
        break;
    case RECORD_STATE_RUN:
        if (!longPress) {
            Model.PlayMusic("UnstableConnect");
            Model.RecorderCommand(Model.REC_PAUSE);
            SetBtnRecImgSrc("start");
            recState = RECORD_STATE_PAUSE;
        }
        break;
    case RECORD_STATE_PAUSE:
        if (longPress)  { /* → STOP */ Model.RecorderCommand(Model.REC_READY_STOP); recState = RECORD_STATE_STOP; }
        else            { /* → RUN  */ Model.RecorderCommand(Model.REC_CONTINUE);   recState = RECORD_STATE_RUN;  }
        break;
    case RECORD_STATE_STOP:
        if (longPress)  { /* → READY */ Model.RecorderCommand(Model.REC_STOP);     recState = RECORD_STATE_READY; }
        else            { /* → RUN   */ Model.RecorderCommand(Model.REC_CONTINUE); recState = RECORD_STATE_RUN;   }
        break;
    }
}

SD卡地图文件结构

地图以预先切片的瓦片（Tile）形式离线存储在 SD 卡上。每张瓦片是 256×256 像素的图片文件，按"缩放级别 / 列号 / 行号"的三级目录组织。

SD卡根目录
└── MAP/                          ← 地图根目录（CONFIG_MAP_DIR_PATH_DEFAULT = "/MAP"）
     └── {缩放级别}/               ← 整数，0~19，默认 16
          └── {tileX}/            ← 瓦片列号 = pixelX / 256
               └── {tileY}.bin   ← 瓦片行号 = pixelY / 256

GPS坐标转换流水线

GPS 模块输出 WGS-84 标准经纬度。地图显示需经过最多三步转换，最终得到用于构造 SD 卡文件路径的像素坐标。整个流程在 MapConv::ConvertMapCoordinate()（MapConv.cpp:57）中串联。

1
GPS 硬件输出 — WGS-84 经纬度
GPS 模块通过 UART2（PA2/PA3，波特率可配）持续输出 NMEA-0183 格式语句（GPRMC、GPGGA 等）。TinyGPSPlus 库实时解析语句，DP_GPS 模块在卫星数 ≥ 3 时将数据发布到 DataCenter。LiveMapModel::GetGPS_Info() 通过 Account::Pull("GPS", &info) 主动拉取；若 GPS 无效（isVaild == false），自动回退到系统配置的默认坐标。
输出：wgLat（WGS-84纬度°）, wgLon（WGS-84经度°）
HAL_GPS.cpp → TinyGPSPlus → DP_GPS.cpp → Account::Pull("GPS")
2
WGS-84 → GCJ-02 偏移纠偏（可选，中国大陆区域地图）
中国法规要求国内地图服务（高德、腾讯等）使用 GCJ-02 坐标系（俗称"火星坐标"），其与 WGS-84 存在随机偏移（在中国大陆约 100~700 米）。通过 MapConv::SetCoordTransformEnable(true) 启用此步骤；使用 OpenStreetMap 等 WGS-84 地图时应关闭。

void TileConv::SetFocusPos(int32_t x, int32_t y)
{
    priv.pointFocus = {x, y};   // 记录焦点（GPS全局像素坐标）

    // ① 计算视口四角（全局像素坐标）
    priv.pointView[0] = {x - viewHalfWidth,  y - viewHalfHeight};  // 左上
    priv.pointView[1] = {x + viewHalfWidth,  y - viewHalfHeight};  // 右上
    priv.pointView[2] = {x - viewHalfWidth,  y + viewHalfHeight};  // 左下
    priv.pointView[3] = {x + viewHalfWidth,  y + viewHalfHeight};  // 右下

    // ② 计算瓦片容器尺寸（额外多2个瓦片，防止边缘露白）
    int32_t tileContW = (viewWidth  / 256 + 2) * 256;  // 240→768，320→512
    int32_t tileContH = (viewHeight / 256 + 2) * 256;

    // ③ 容器左上角：视口左上角所在位置向下对齐到256的整数倍边界
    priv.pointTileCont[0].x = FixTile(priv.pointView[0].x, false);  // false=向下取整
    priv.pointTileCont[0].y = FixTile(priv.pointView[0].y, false);
    // 其余三角 = 左上角坐标 + 容器固定尺寸
    priv.pointTileCont[1] = {tileContLeft + tileContW, tileContTop};
    priv.pointTileCont[2] = {tileContLeft, tileContTop + tileContH};
    priv.pointTileCont[3] = {tileContLeft + tileContW, tileContTop + tileContH};
}

// FixTile()：将坐标对齐到256整数倍边界
int32_t TileConv::FixTile(int32_t x, bool up)
{
    int32_t r = x % 256;
    int32_t ret = x - r;       // 向下取整（去掉余数）
    if (up) ret += 256;        // 向上取整
    return ret;
}

LVGL 折线的点坐标是相对于瓦片容器 LVGL 对象（ui.map.cont）的。因此必须将全局像素坐标转换为容器内偏移（LiveMap.cpp:372）：

void LiveMap::TrackLineAppendToEnd(int32_t x, int32_t y)
{
    TileConv::Point_t offset;
    TileConv::Point_t curPoint = {x, y};
    Model.tileConv.GetOffset(&offset, &curPoint);
    // GetOffset(): offset = (point.x - tileContLeft, point.y - tileContTop)
    // 即：GPS全局像素坐标 减去 瓦片容器左上角坐标 = 容器内相对坐标

    View.ui.track.lineTrack->append_to_end(
        (lv_coord_t)offset.x,
        (lv_coord_t)offset.y);
}

缩放级别切换时的历史轨迹坐标重算

历史轨迹点以某一缩放级别的全局像素坐标存储（通过 PointContainer 压缩存储差分坐标）。切换缩放级别时，所有历史点的坐标必须按新级别重新换算（MapConv.cpp:81）：

// 原理：mapSize = 256 × 2^level，级别每增加1，全局像素坐标翻倍
// 因此不同级别间的换算可用位移运算代替浮点乘除
void MapConv::ConvertMapLevelPos(int32_t* destX, int32_t* destY,
                                  int32_t srcX, int32_t srcY, int srcLevel)
{
    int diffLevel = srcLevel - GetLevel();
    if (diffLevel >= 0)
        { *destX = srcX >> diffLevel; *destY = srcY >> diffLevel; }  // 缩小：右移
    else
        { *destX = srcX << -diffLevel; *destY = srcY << -diffLevel; }  // 放大：左移
}

// 调用位置（LiveMapModel.cpp:111）：
// TrackReload() 遍历 PointContainer 时，对每个存储点调用此函数
// 将记录时的级别(info.level) → 当前显示级别(mapConv.GetLevel()) 转换
